# 12장 제품 소프트웨어 패키징 — 예상 문제

---

## 유형 1. 설명 → 용어 맞추기

> 설명을 읽고 해당하는 용어를 쓰시오.

**Q01.**  
개발 과정에서 소프트웨어의 변경 사항을 관리하기 위해 개발된 일련의 활동으로, 소프트웨어 개발의 전 단계에 적용되며 대표적인 도구에는 Git, SVN, CVS 등이 있다.

**Q02.**  
형상 관리 기능 중, 식별된 형상 항목에 대한 변경 요구를 검토하여 현재의 기준선(Base Line)이 잘 반영될 수 있도록 조정하는 작업이다.

**Q03.**  
소프트웨어의 버전 등록 관련 기능 중, 프로그램을 수정하기 위해 저장소(Repository)에서 파일을 받아오는 것이다.

**Q04.**  
소프트웨어의 버전 등록 관련 기능 중, 체크아웃 한 파일의 수정을 완료한 후 저장소(Repository)의 파일을 새로운 버전으로 갱신하는 것이다.

**Q05.**  
빌드를 포함하여 테스트 및 배포를 자동화하는 도구로, Jenkins, Gradle, Ant, Maven, Make 등이 있다.

## 유형 2. 핵심 빈칸 뚫기

**Q06.**  
형상 관리의 기능 5가지를 쓰시오.

( ① ), ( ② ), ( ③ ), ( ④ ), ( ⑤ )

**Q07.**  
소프트웨어 버전 등록 관련 주요 기능을 설명에 맞게 쓰시오.

① 최신 버전의 파일들과 변경 내역이 저장되어 있는 곳 → ( ① )  
② 저장소에 처음으로 파일을 복사함 → ( ② )  
③ 저장소에서 파일을 받아옴 → ( ③ )  
④ 수정을 완료한 후 저장소를 새로운 버전으로 갱신함 → ( ④ )  
⑤ 저장소에 있는 최신 버전으로 자신의 작업 공간을 동기화함 → ( ⑤ )
