# 2장 데이터 입·출력 구현 — 예상 문제 (p.16~31)

---

## 유형 1. 설명 → 용어 맞추기

> 설명을 읽고 해당하는 용어를 쓰시오.

**Q01.**  
스키마의 종류 중, 사용자나 응용 프로그래머가 각 개인의 입장에서 필요로 하는 데이터베이스의 논리적 구조를 정의한 것이다.

**Q02.**  
스키마의 종류 중, 데이터베이스의 전체적인 논리적 구조로, 모든 응용 프로그램이나 사용자들이 필요로 하는 데이터를 종합한 것이다. 하나만 존재한다.

**Q03.**  
스키마의 종류 중, 물리적 저장장치의 입장에서 본 데이터베이스 구조로, 실제로 저장될 레코드 형식·저장 데이터 항목의 표현 방법 등을 나타낸다.

**Q04.**  
데이터베이스 설계 5단계 중, 현실 세계의 정보를 DBMS에 독립적인 E-R 다이어그램으로 표현하는 단계이다. 개념 스키마 모델링과 트랜잭션 모델링을 병행 수행한다.

**Q05.**  
데이터베이스 설계 5단계 중, 특정 DBMS가 지원하는 논리적 자료 구조로 변환(mapping)시키는 단계이다. 개념 스키마를 평가·정제하고 논리 스키마를 설계한다.

**Q06.**  
데이터베이스 설계 5단계 중, 저장 구조 및 액세스 경로를 결정하는 단계이다. 물리적 저장장치에 저장할 수 있는 물리 구조의 데이터로 변환하는 과정이다.

**Q07.**  
릴레이션의 각 행(Row)을 의미하며, 속성의 모임으로 구성된다. 파일 구조에서 레코드에 해당하며, 그 수를 카디널리티(Cardinality)라고 한다.

**Q08.**  
릴레이션의 각 열(Column)에 해당하며, 데이터베이스를 구성하는 가장 작은 논리적 단위이다. 그 수를 디그리(Degree) 또는 차수라고 한다.

**Q09.**  
하나의 속성(Attribute)이 취할 수 있는 같은 타입의 원자(Atomic)값들의 집합이다. 예를 들어 '성별' 속성의 경우 "남"과 "여"가 해당한다.

**Q10.**  
릴레이션을 구성하는 속성들 중 튜플을 유일하게 식별하기 위해 사용되는 속성들의 부분집합이다. **유일성(Unique)과 최소성(Minimality)을 모두 만족**해야 한다.

**Q11.**  
후보키가 둘 이상일 때 기본키를 제외한 나머지 후보키를 의미한다. 보조키라고도 한다.

**Q12.**  
릴레이션을 구성하는 속성들의 집합으로 구성된 키로, **유일성은 만족하지만 최소성은 만족하지 못하는** 키이다.

**Q13.**  
다른 릴레이션의 기본키를 참조하는 속성 또는 속성들의 집합이다. 참조 릴레이션의 기본키에 없는 값은 입력할 수 없다.

**Q14.**  
데이터베이스 무결성의 종류 중, **기본 테이블의 기본키를 구성하는 어떤 속성도 Null 값이나 중복값을 가질 수 없다는** 규정이다.

**Q15.**  
데이터베이스 무결성의 종류 중, **외래키 값은 Null이거나 참조 릴레이션의 기본키 값과 동일해야 한다는** 규정이다.

**Q16.**  
관계형 데이터베이스에서 원하는 정보와 그 정보를 검색하기 위해 어떻게 유도하는가를 기술하는 **절차적인 언어**이다. 연산자와 연산규칙을 제공하며, 피연산자와 연산 결과가 모두 릴레이션이다.

**Q17.**  
순수 관계 연산자 중, 릴레이션에서 선택 조건을 만족하는 **튜플의 부분집합**을 구하는 연산이다. 수평 연산이라고도 하며, 기호는 σ(시그마)이다.

**Q18.**  
순수 관계 연산자 중, 릴레이션에서 제시된 **속성 값만을 추출**하여 새로운 릴레이션을 만드는 연산이다. 수직 연산자라고도 하며, 기호는 π(파이)이다.

**Q19.**  
관계 데이터의 연산을 표현하는 방법으로, 코드(E. F. Codd)가 수학의 술어 해석(Predicate Calculus)에 기반하여 제안했다. **원하는 정보가 무엇인지만 정의하는 비절차적** 특성을 지닌다.

**Q20.**  
데이터베이스 내에 데이터들이 불필요하게 중복되어 릴레이션 조작 시 예기치 않게 발생하는 곤란한 현상이다. 삽입 이상·삭제 이상·갱신 이상 세 종류가 있다.

**Q21.**  
함수적 종속의 종류 중, 속성 Y가 기본키 X **전체**에 대해 함수적 종속이면서 X의 어떠한 진부분 집합에도 함수적 종속이 아닐 때를 말한다. 2NF에서 제거 대상이 된다.

**Q22.**  
함수적 종속의 종류 중, X → Y이고 Y → Z일 때 X → Z를 만족하는 관계이다. 3NF에서 제거 대상이 된다.

**Q23.**  
테이블 R에 속한 모든 속성의 도메인이 **원자 값(Atomic Value)만으로** 되어 있는 정규형이다. 반복 그룹이 존재하지 않는 상태이다.

**Q24.**  
정규화된 데이터 모델을 시스템의 성능 향상과 개발·운영의 편의성을 위해 **의도적으로 통합, 중복, 분리하여 정규화 원칙을 위배**하는 행위이다.

**Q25.**  
시스템 그 자체에 관련이 있는 다양한 객체에 관한 정보를 포함하는 시스템 데이터베이스이다. **데이터 사전(Data Dictionary)**이라고도 하며, 사용자는 조회만 가능하다.

**Q26.**  
트랜잭션 특성(ACID) 중, 트랜잭션의 연산이 데이터베이스에 **모두 반영(Commit)되든지 전혀 반영되지 않도록(Rollback)** 되어야 한다는 특성이다.

**Q27.**  
트랜잭션 특성(ACID) 중, 둘 이상의 트랜잭션이 동시에 병행 실행되는 경우 **어느 하나의 트랜잭션 실행 중에 다른 트랜잭션의 연산이 끼어들 수 없다**는 특성이다.

**Q28.**  
사용자에게 접근이 허용된 자료만을 제한적으로 보여주기 위해 하나 이상의 기본 테이블로부터 유도된, 이름을 가지는 **가상 테이블**이다. 정의할 때는 CREATE문, 제거할 때는 DROP문을 사용한다.

**Q29.**  
분산 데이터베이스의 목표 중, 액세스하려는 데이터베이스의 **실제 위치를 알 필요 없이** 단지 논리적인 명칭만으로 액세스할 수 있는 특성이다.

**Q30.**  
비상사태 또는 업무 중단 시점으로부터 **복구되어 가동될 때까지의 소요 시간**을 의미하는 용어이다.

**Q31.**  
비상사태 또는 업무 중단 시점으로부터 **데이터를 복구할 수 있는 기준점**을 의미하는 용어이다.

**Q32.**  
접근통제 방식 중, **데이터에 접근하는 사용자의 신원에 따라** 데이터 소유자가 접근 권한을 지정하고 제어하는 방식이다.

**Q33.**  
접근통제 방식 중, **주체와 객체의 등급을 비교하여 시스템이** 접근 권한을 지정하는 방식이다. 데이터베이스 객체별로 보안 등급을 부여할 수 있다.

**Q34.**  
접근통제 방식 중, **사용자의 역할에 따라 중앙관리자가** 접근 권한을 부여하는 방식이다. 임의 접근통제와 강제 접근통제의 단점을 보완하였다.

**Q35.**  
자료 구조 중, 리스트의 한쪽 끝으로만 자료의 삽입·삭제 작업이 이루어지는 자료구조이다. **후입선출(LIFO)** 방식으로 처리하며, 꽉 찬 상태에서 삽입 시 오버플로가 발생한다.

**Q36.**  
이진 트리 운행법 중, **Root → Left → Right** 순으로 노드들을 방문하는 방법이다.

**Q37.**  
이진 트리 운행법 중, **Left → Root → Right** 순으로 노드들을 방문하는 방법이다. 이진탐색트리에서는 오름차순 출력 결과를 얻을 수 있다.

**Q38.**  
n개의 레코드 중에서 **최소값을 찾아 첫 번째 레코드 위치에 놓는 방식**을 반복하는 정렬이다. 평균과 최악 모두 O(n²)이다.

**Q39.**  
주어진 파일에서 **인접한 두 개의 레코드 키 값을 비교하여 그 크기에 따라 서로 교환**하는 정렬 방식이다. 평균과 최악 모두 O(n²)이다.

**Q40.**  
**키를 기준으로 작은 값은 왼쪽, 큰 값은 오른쪽 서브 파일에 분해**시키는 과정을 반복하는 정렬 방식이다. 평균 O(nlog₂n), 최악 O(n²)이다.
