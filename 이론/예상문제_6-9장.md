# 6~9장 예상 문제 (p.7~10)

---

## 유형 1. 설명 → 용어 맞추기

**Q01.**  
사용자와 시스템 간의 상호작용이 원활하게 이뤄지도록 도와주는 장치나 소프트웨어로, 사용성·접근성·편의성을 중시한다.

**Q02.**  
사용자가 시스템이나 서비스를 이용하면서 느끼고 생각하게 되는 총체적인 경험을 의미하며, UI를 통해 사용자가 느끼는 만족이나 감정을 중시한다.

**Q03.**  
UI의 구분 중, 명령과 출력이 텍스트 형태로 이뤄지는 인터페이스이다.

**Q04.**  
UI의 구분 중, 아이콘이나 메뉴를 마우스로 선택하여 작업을 수행하는 그래픽 환경의 인터페이스이다.

**Q05.**  
UI의 구분 중, 사용자의 말이나 행동 등 자연스러운 움직임을 통해 기기를 조작하는 인터페이스이다.

**Q06.**  
UI의 기본 원칙 중, 사용자의 요구사항을 최대한 수용하고 실수를 최소화해야 하는 원칙이다.

**Q07.**  
ISO/IEC 9126 소프트웨어 품질 특성 중, 환경의 변화 또는 새로운 요구사항이 발생했을 때 소프트웨어를 개선하거나 확장할 수 있는 정도를 나타낸다. 하위 특성으로 분석성, 변경성, 안정성, 시험성이 있다.

**Q08.**  
ISO/IEC 9126 소프트웨어 품질 특성 중, 소프트웨어가 다른 환경에서도 얼마나 쉽게 적용할 수 있는지 나타낸다. 하위 특성으로 적용성, 설치성, 대체성, 공존성이 있다.

**Q09.**  
애플리케이션 테스트의 기본 원리 중, 동일한 테스트 케이스로 반복 테스트를 수행하면 더 이상 결함이 발견되지 않는 현상이다.

**Q10.**  
애플리케이션 테스트의 기본 원리 중, 소프트웨어의 결함을 모두 제거해도 사용자의 요구사항을 만족시키지 못하면 해당 소프트웨어는 품질이 높다고 말할 수 없는 것이다.

**Q11.**  
프로그램을 실행하지 않고 명세서나 소스 코드를 대상으로 분석하는 테스트로, 워크스루·인스펙션·코드 검사 등이 있다.

**Q12.**  
소프트웨어가 수행할 특정 기능을 알기 위해 각 기능이 완전히 작동되는 것을 입증하는 테스트로, 기능 테스트라고도 한다. 사용자의 요구사항 명세를 보면서 테스트한다.

**Q13.**  
블랙박스 테스트 기법 중, 입력 조건의 경계값에서 오류가 발생될 확률이 높다는 점을 이용하여 입력 조건의 경계값을 테스트 케이스로 선정하여 검사하는 기법이다.

**Q14.**  
블랙박스 테스트 기법 중, 입력 데이터 간의 관계와 출력에 영향을 미치는 상황을 체계적으로 분석한 다음 효용성이 높은 테스트 케이스를 선정하여 검사하는 기법이다.

**Q15.**  
코딩 직후 소프트웨어 설계의 최소 단위인 모듈이나 컴포넌트에 초점을 맞춰 테스트하는 것이다.

**Q16.**  
단위 테스트가 완료된 모듈들을 결합하여 하나의 시스템으로 완성시키는 과정에서의 테스트로, 모듈 간 상호작용 오류를 검사한다.

**Q17.**  
통합 테스트 방식 중, 단계적으로 통합하는 절차 없이 모든 모듈이 미리 결합되어 있는 프로그램 전체를 테스트하는 방식이다.

**Q18.**  
개발한 소프트웨어가 사용자의 요구사항을 충족하는지에 중점을 두고, 사용자가 직접 테스트하는 것이다.

**Q19.**  
인수 테스트 중, 개발자의 장소에서 사용자가 개발자 앞에서 행하는 테스트 기법으로, 통제된 환경에서 오류와 사용상의 문제점을 함께 확인하면서 기록한다.

**Q20.**  
통합 테스트로 인해 변경된 모듈이나 컴포넌트에 새로운 오류가 있는지 확인하기 위해 이미 테스트된 프로그램의 테스팅을 반복하는 테스트이다.

**Q21.**  
하향식 통합 테스트에서, 제어 모듈이 호출하는 타 모듈의 기능을 단순히 수행하는 일시적인 시험용 모듈이다.

**Q22.**  
상향식 통합 테스트에서, 테스트 대상의 하위 모듈을 호출하고 파라미터를 전달하여 모듈 테스트 수행 후의 결과를 도출하는 도구이다.

**Q23.**  
테스트 결과가 올바른지 판단하기 위해 사전에 정의된 참값을 대입하여 비교하는 기법 및 활동이다.

**Q24.**  
한 프로그램의 논리적인 복잡도를 측정하기 위한 소프트웨어의 척도로, 맥케이브 순환도라고도 한다. V(G) = E − N + 2 공식으로 계산할 수 있다.

**Q25.**  
소스 코드에서 코드의 로직이 서로 복잡하게 얽혀 있는 코드를 의미하는 용어이다.

**Q26.**  
아주 오래되거나 참고문서 또는 개발자가 없어 유지보수 작업이 어려운 코드를 뜻하는 용어이다.

**Q27.**  
소스 코드 품질 분석 도구 중, 작성한 소스 코드를 실행하여 코드에 존재하는 메모리 누수, 스레드 결함 등을 분석하는 도구로, Avalanche·Valgrind 등이 있다.

**Q28.**  
DDL 명령어 중, DROP 사용 시 제거할 요소를 참조하는 다른 모든 개체를 함께 제거하는 옵션이다.

**Q29.**  
DDL 명령어 중, DROP 사용 시 다른 개체가 제거할 요소를 참조 중일 때 제거를 취소하는 옵션이다.

**Q30.**  
DB 구조, 데이터 형식, 접근 방식 등 DB를 구축하거나 수정할 목적으로 사용하는 언어로, CREATE·ALTER·DROP이 있다.

**Q31.**  
데이터의 보안, 무결성, 회복, 병행 제어 등을 정의하는 데 사용하는 언어로, DBA가 데이터 관리를 목적으로 사용하며 COMMIT·ROLLBACK·GRANT·REVOKE가 있다.

**Q32.**  
데이터베이스 사용자가 저장된 데이터를 실질적으로 관리하는 데 사용하는 언어로, SELECT·INSERT·DELETE·UPDATE가 있다.

**Q33.**  
변경되었으나 아직 COMMIT되지 않은 모든 내용을 취소하고 데이터베이스를 이전 상태로 되돌리는 명령어이다.

**Q34.**  
SQL에서 GROUP BY절과 함께 사용되며, 그룹에 대한 조건을 지정하는 절이다.

**Q35.**  
두 개 이상의 테이블을 통합할 때 사용하는 집합 연산자 중, 두 SELECT문의 조회 결과를 통합하여 모두 출력하되 중복된 행은 한 번만 출력하는 연산자이다.

**Q36.**  
두 개의 SELECT문 조회 결과 중 공통된 행만 출력하는 집합 연산자이다. (교집합)

**Q37.**  
JOIN의 종류 중, EQUI JOIN에서 JOIN 조건이 '='일 때 동일한 속성이 두 번 나타나게 되는 중복 속성을 제거하여 같은 속성을 한 번만 표기하는 조인 방법이다.

**Q38.**  
데이터베이스 시스템에서 데이터의 삽입·갱신·삭제 등의 이벤트가 발생할 때 관련 작업이 자동으로 수행되게 하는 절차형 SQL이다.

**Q39.**  
소프트웨어 개발 보안의 3대 요소 중, 시스템 내의 정보와 자원은 인가된 사용자에게만 접근이 허용되는 것이다.

**Q40.**  
소프트웨어 개발 보안의 3대 요소 중, 인가받은 사용자는 시스템 내의 정보와 자원을 언제라도 사용할 수 있는 것이다.

**Q41.**  
데이터를 송·수신한 자가 송·수신 사실을 부인할 수 없도록 송·수신 증거를 제공하는 보안 기술이다.

**Q42.**  
웹 응용 프로그램에 SQL을 삽입하여 내부 DB 서버의 데이터를 유출·변조하고 관리자 인증을 우회하는 보안 약점이다.

**Q43.**  
동일한 키로 데이터를 암호화하고 복호화하는 암호화 기법으로, 대칭 암호 기법 또는 단일키 암호화 기법이라고도 한다. 암호화·복호화 속도가 빠르지만 관리해야 할 키의 수가 많다.

**Q44.**  
2001년 미국 NIST에서 발표한 개인키 암호화 알고리즘으로, 블록 크기는 128비트이며 키 길이에 따라 AES-128/192/256으로 분류된다.

**Q45.**  
임의의 길이의 입력 데이터를 고정된 길이의 값이나 키로 변환하는 것으로, SHA 시리즈·MD5·HAVAL 등이 있다.

**Q46.**  
인터넷 애플리케이션에서 사용자 인증에 사용되는 표준 인증 방법으로, 공개 API로 구현되어 비밀번호 제공 없이 접근 권한을 부여하여 사용할 수 있다.

**Q47.**  
인터넷 등 통신 사업자의 공중 네트워크와 암호화 기술을 이용하여 사용자가 마치 자신의 전용 회선을 사용하는 것처럼 해주는 보안 솔루션이다.

**Q48.**  
다양한 장비에서 발생하는 로그 및 보안 이벤트를 통합하여 관리하는 빅데이터 기반의 보안 솔루션이다.

**Q49.**  
다른 컴퓨터에 원격으로 접속하여 작업을 수행할 수 있도록 다양한 기능을 지원하는 프로토콜로, 기본적으로 22번 포트를 사용한다.

**Q50.**  
SYN Flooding 공격에서, 신뢰성 있는 연결을 위한 3단계 확인 작업 절차의 명칭이다.

**Q51.**  
IP나 ICMP의 특성을 악용하여 엄청난 양의 데이터를 한 사이트에 집중적으로 보냄으로써 네트워크를 불능 상태로 만드는 공격 방법이다.

**Q52.**  
상호 인증 과정을 거친 후 접속해 있는 서버와 클라이언트 사이의 세션 정보를 가로채는 공격 기법으로, 인증 정보 없이도 공격자가 클라이언트인 것처럼 위장하여 서버 자원을 무단으로 사용한다.

**Q53.**  
ARP의 취약점을 이용한 공격 기법으로, 자신의 물리적 주소(MAC)를 공격 대상의 것으로 변조하여 데이터 패킷을 가로채거나 방해하는 기법이다.

**Q54.**  
목표 대상이 자주 방문하는 웹 사이트를 사전에 감염시켜 대상이 해당 사이트에 방문했을 때 악성 코드에 감염되게 하는 웹 기반 공격이다.

**Q55.**  
인터넷 사용자의 컴퓨터에 잠입해 내부 문서나 파일 등을 암호화해 사용자가 열지 못하게 하고, 암호 해독용 프로그램의 전달을 조건으로 금전을 요구하는 프로그램이다.

**Q56.**  
네트워크를 통해 연속적으로 자신을 복제하여 시스템의 부하를 높임으로써 시스템을 다운시키는 악성 프로그램으로, 자기 복제 능력이 있다.
