# 1장 요구사항 확인 — 예상 문제 (p.2~5)

---

## 유형 1. 설명 → 용어 맞추기

> 설명을 읽고 해당하는 용어를 쓰시오.

**Q01.**  
소프트웨어 개발 모형 중, **보헴(Boehm)** 이 제안한 것으로 위험 최소화에 초점을 맞춘 모형이다.  
계획 수립 → 위험 분석 → 개발 및 검증 → 고객 평가의 4단계를 반복하며 진행된다.

**Q02.**  
가장 오래된 고전적 소프트웨어 생명주기 모형으로, 각 단계가 순차적으로 진행되며 이전 단계가 완료된 후 다음 단계로 넘어간다. 단계 간 되돌아가기 어렵다는 단점이 있다.

**Q03.**  
"민첩한"이라는 의미를 가진 소프트웨어 개발 방법론으로, 짧은 반복 개발 주기를 통해 변화에 유연하게 대응한다. 대표 모형으로 스크럼(Scrum), XP, 칸반, Lean, FDD 등이 있다.

**Q04.**  
XP(eXtreme Programming)의 5가지 핵심 가치 중 하나로, 고객·팀원 간의 원활한 정보 공유를 의미하며, 짝 프로그래밍과 일일 스탠드업 미팅을 통해 실천된다.

**Q05.**  
XP 실천 방법 중, 두 명의 개발자가 한 컴퓨터를 함께 사용하여 코드를 작성하는 방법이다. 한 명은 코드를 직접 작성하고 다른 한 명은 검토하며 협력한다.

**Q06.**  
XP 실천 방법 중, 코드를 먼저 작성하는 것이 아니라 테스트 케이스를 먼저 작성한 후 이를 통과하는 코드를 작성하는 개발 방법이다.

**Q07.**  
요구사항의 종류 중, 시스템이 "무엇을 해야 하는가"에 대한 요구사항이다. 예를 들어 "로그인 기능을 제공해야 한다"와 같이 시스템이 수행해야 할 기능과 동작을 명세한다.

**Q08.**  
요구사항의 종류 중, 시스템이 "어떠해야 하는가"에 대한 요구사항이다. 성능, 보안, 신뢰성, 유지보수성 등 시스템의 품질 속성을 명세한다.

**Q09.**  
자료 흐름도(DFD)의 4가지 구성 요소 중, 입력 자료를 받아 처리하여 출력 자료로 변환하는 요소로, 원(○)으로 표현된다.

**Q10.**  
자료 흐름도(DFD)의 4가지 구성 요소 중, 시스템 외부의 입력 또는 출력의 시작점/종점으로, 사각형(□)으로 표현된다.

**Q11.**  
시스템 설계 도구 중, 계층(Hierarchy) + 입력(Input) · 처리(Process) · 출력(Output)의 합성어로, 시스템의 기능을 계층적으로 분석하고 표현하는 방법이다.

**Q12.**  
UML 관계 중, 클래스 간 "사용하는" 관계를 나타내며, 한 클래스가 다른 클래스를 일시적으로 참조(파라미터, 반환값, 지역변수 등)할 때 사용된다. 점선 화살표(---▶)로 표현된다.

**Q13.**  
UML 관계 중, 하나의 클래스가 다른 클래스의 부분(part)으로 구성되는 관계이다. 전체와 부분이 독립적으로 생명 주기를 가지는 경우(약한 결합)와 전체가 소멸되면 부분도 소멸되는 경우(강한 결합)로 나뉜다.

**Q14.**  
UML 관계 중, 상위 클래스(부모)와 하위 클래스(자식) 간의 "is-a" 관계를 나타내며, 상속 개념에 해당한다. 속이 빈 삼각형 화살표(──▷)로 표현된다.

**Q15.**  
UML 다이어그램의 두 가지 분류 중, 시스템의 정적 구조를 표현하며 클래스 다이어그램, 객체 다이어그램, 컴포넌트 다이어그램, 배치 다이어그램 등이 포함되는 종류는?

**Q16.**  
UML 행위 다이어그램 중, 액터(Actor)와 시스템 간의 상호작용을 표현하며, 시스템이 제공하는 기능(Use Case)을 사용자 관점에서 보여주는 다이어그램이다.

**Q17.**  
UML 행위 다이어그램 중, 객체 간의 메시지 송수신 순서를 시간 흐름에 따라 수직으로 표현하는 다이어그램이다.

## 유형 2. 핵심 빈칸 뚫기

> 빈칸에 알맞은 용어를 쓰시오.

**Q18.**  
나선형 모형은 ( ① ) 이 제안한 모형으로, ( ② ) 최소화에 초점을 맞춘다.  
4단계 순서는 **계획 수립 → ( ③ ) → 개발 및 검증 → ( ④ )** 이다.

**Q19.**  
애자일 모형의 4가지 핵심 가치는 다음과 같다.  
① 프로세스와 도구보다 ( ㄱ ) 을 중시한다.  
② 포괄적인 문서보다 ( ㄴ ) 하는 소프트웨어를 중시한다.  
③ 계약 협상보다 ( ㄷ ) 과의 협력을 중시한다.  
④ 계획을 따르기보다 ( ㄹ ) 에 반응하는 것을 중시한다.

**Q20.**  
XP의 5가지 핵심 가치는 **의사소통, ( ① ), 용기, ( ② ), 피드백** 이다.

**Q21.**  
XP의 주요 실천 방법 중 다음 설명에 해당하는 용어를 쓰시오.  
- "소스코드는 팀 전원이 소유하고 누구든지 수정할 수 있다" → ( ① )  
- "작은 단위로 자주 배포한다" → ( ② )  
- "코드의 기능 변경 없이 내부 구조를 개선한다" → ( ③ )

**Q22.**  
자료 흐름도(DFD)의 4가지 구성 요소를 모두 쓰시오.

( ① ), ( ② ), ( ③ ), ( ④ )

**Q23.**  
HIPO는 ( ① )(계층) + ( ② )(입력) · ( ③ )(처리) · ( ④ )(출력)의 합성어이다.

**Q24.**  
UML 관계의 종류 4가지를 쓰시오.

( ① ), ( ② ), ( ③ ), ( ④ )

**Q25.**  
UML 구조적 다이어그램의 종류 6가지를 쓰시오.

( ① ), ( ② ), ( ③ ), ( ④ ), ( ⑤ ), ( ⑥ )

**Q26.**  
UML 행위 다이어그램의 종류 7가지를 쓰시오.

( ① ), ( ② ), ( ③ ), ( ④ ), ( ⑤ ), ( ⑥ ), ( ⑦ )

**Q27.**  
요구사항의 두 가지 종류를 쓰고, 각각 한 줄로 설명하시오.

**Q28.**  
XP 실천 방법 중 테스트 케이스를 먼저 작성한 후 코드를 작성하는 방법을 ( ① )이라 하고, 팀의 모든 구성원이 개발에 참여하는 방법을 ( ② )이라 한다.
