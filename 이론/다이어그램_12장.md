## 버전 등록 주요 기능 (298) ★B — 작업 흐름

```mermaid
sequenceDiagram
    participant DEV as 👤 개발자
    participant LOCAL as 💻 로컬 작업 공간
    participant REPO as 🗄️ 저장소 Repository

    Note over REPO: 최신 버전 파일 + 변경 내역 저장

    DEV->>REPO: ① 가져오기 (Import)<br>처음으로 파일 복사

    REPO-->>LOCAL: ② 체크아웃 (Check-Out)<br>수정을 위해 파일 받아옴

    DEV->>LOCAL: 파일 수정 작업

    LOCAL->>REPO: ③ 체크인 (Check-In)<br>수정 완료 → 새 버전으로 갱신

    Note over LOCAL,REPO: 충돌(Conflict) 발생 시<br>④ 커밋 (Commit)<br>diff 도구로 수정 후 갱신 완료

    REPO-->>LOCAL: ⑤ 동기화 (Update)<br>최신 버전으로 작업 공간 동기화
```
