## [5장 인터페이스 구현] 유형 1. 설명 → 용어 맞추기

> 설명을 읽고 해당하는 용어를 쓰시오.

**Q01.**  
요구사항 검증 방법 중, **요구사항 명세서 작성자가 명세서 내용을 직접 설명하고 동료들이 이를 들으면서 결함을 발견**하는 형태의 검토 방법이다.

**Q02.**  
요구사항 검증 방법 중, **검토 회의 전에 요구사항 명세서를 미리 배포하여 사전 검토한 후에 짧은 검토 회의를 통해 결함을 발견**하는 형태의 검토 방법이다.

**Q03.**  
요구사항 검증 방법 중, **요구사항 명세서 작성자를 제외한 다른 검토 전문가들이** 요구사항 명세서를 확인하면서 결함을 발견하는 형태의 검토 방법이다.

**Q04.**  
운영체제와 응용 프로그램, 또는 서버와 클라이언트 사이에서 **다양한 서비스를 제공하는 소프트웨어**이다. 표준화된 인터페이스를 제공함으로써 시스템 간의 데이터 교환에 일관성을 보장한다.

**Q05.**  
미들웨어의 종류 중, **메시지 기반의 비동기형 메시지를 전달**하는 미들웨어이다. 온라인 업무보다는 이기종 분산 데이터 시스템의 데이터 동기를 위해 많이 사용된다.

**Q06.**  
미들웨어의 종류 중, **온라인 트랜잭션 업무에서 트랜잭션을 처리 및 감시**하는 미들웨어이다. 항공기·철도 예약 업무 등 빠른 응답 속도를 유지해야 하는 업무에 주로 사용된다.

**Q07.**  
미들웨어의 종류 중, **사용자의 요구에 따라 변하는 동적인 콘텐츠를 처리**하기 위한 미들웨어이다. 웹 환경을 구현하기 위한 미들웨어로, Java·EJB 컴포넌트 기반으로 구현 가능하다.

**Q08.**  
**기업 내 각종 애플리케이션 및 플랫폼 간의 정보 전달, 연계, 통합 등 상호 연동**이 가능하게 해주는 솔루션이다. 연동 방식으로 Point-to-Point, Hub & Spoke, Message Bus, Bus Hybrid 등이 있다.

**Q09.**  
EAI 연동 방식 중, **단일 접점인 허브 시스템을 통해 데이터를 전송하는 중앙 집중형 방식**이다. 확장 및 유지 보수가 용이하지만, 허브 장애 발생 시 시스템 전체에 영향을 미친다.

**Q10.**  
EAI 연동 방식 중, **Hub & Spoke와 Message Bus의 혼합 방식**이다. 그룹 내에서는 Hub & Spoke 방식을, 그룹 간에는 Message Bus 방식을 사용한다.

**Q11.**  
웹과 컴퓨터 프로그램에서 용량이 적은 데이터를 교환하기 위해 **데이터 객체를 속성·값의 쌍(Attribute-Value Pairs) 형태로 표현하는 개방형 표준 포맷**이다. 비동기 처리에 사용되는 AJAX에서 XML을 대체하여 사용되고 있다.

**Q12.**  
자바 스크립트(JavaScript)를 사용하여 클라이언트와 서버 간에 XML 데이터를 주고 받는 **비동기 통신 기술**이다. 전체 페이지를 새로 고치지 않고도 웹 페이지 일부 영역만을 업데이트할 수 있다.

**Q13.**  
인터페이스 보안에서 사용되는 네트워크 보안 프로토콜 중, **네트워크 계층에서 IP 패킷 단위의 데이터 변조 방지 및 은닉 기능을 제공**하는 프로토콜이다. 암호화와 복호화가 모두 가능한 양방향 암호화 방식을 사용한다.

**Q14.**  
인터페이스 보안에서 사용되는 네트워크 보안 프로토콜 중, **TCP/IP 계층과 애플리케이션 계층 사이에서 인증, 암호화, 무결성을 보장**하는 프로토콜이다.

**Q15.**  
인터페이스 구현 검증 도구 중, **같은 테스트 코드를 여러 번 작성하지 않게 도와주며, 테스트마다 예상 결과를 기억할 필요가 없게 하는 단위 테스트 프레임워크**이다. Java의 JUnit, C++의 CppUnit 등으로 불리며 통합하여 이 이름으로 통칭된다.

**Q16.**  
인터페이스 구현 검증 도구 중, **서비스 호출 및 컴포넌트 재사용 등 다양한 환경을 지원하는 테스트 프레임워크**이다. 크로스 플랫폼이나 분산 소프트웨어에서 테스트 환경을 조성할 수 있도록 지원한다.

**Q17.**  
인터페이스 구현 검증 도구 중, **다양한 브라우저 및 개발 언어를 지원하는 웹 애플리케이션 테스트 프레임워크**이다.

**Q18.**  
인터페이스 구현 검증 도구 중, **FitNesse의 장점인 협업 기능과 STAF의 장점인 재사용 및 확장성을 통합**한 NHN(Naver)의 테스트 자동화 프레임워크이다.

## [6장 화면 설계] 유형 1. 설명 → 용어 맞추기

**Q25.**  
사용자와 시스템 간의 상호작용이 원활하게 이뤄지도록 도와주는 장치나 소프트웨어로, 사용성·접근성·편의성을 중시한다.

**Q26.**  
사용자가 시스템이나 서비스를 이용하면서 느끼고 생각하게 되는 총체적인 경험을 의미하며, UI를 통해 사용자가 느끼는 만족이나 감정을 중시한다.

**Q27.**  
UI의 구분 중, 명령과 출력이 텍스트 형태로 이뤄지는 인터페이스이다.

**Q28.**  
UI의 구분 중, 아이콘이나 메뉴를 마우스로 선택하여 작업을 수행하는 그래픽 환경의 인터페이스이다.

**Q29.**  
UI의 구분 중, 사용자의 말이나 행동 등 자연스러운 움직임을 통해 기기를 조작하는 인터페이스이다.

**Q30.**  
UI의 기본 원칙 중, 사용자의 요구사항을 최대한 수용하고 실수를 최소화해야 하는 원칙이다.
