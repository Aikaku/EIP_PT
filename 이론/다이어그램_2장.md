## DB 설계 5단계 ★A

```mermaid
sequenceDiagram
    participant REQ as 요구사항 분석
    participant CON as 개념적 설계
    participant LOG as 논리적 설계
    participant PHY as 물리적 설계
    participant IMP as 구현

    Note over REQ: 업무 분석\n데이터 요구사항 수집

    REQ->>CON: 요구사항 명세서
    Note over CON: E-R 다이어그램 작성\n개념 스키마 설계\nDBMS 독립적

    CON->>LOG: E-R 다이어그램
    Note over LOG: 릴레이션 스키마 변환\n정규화 수행\nDBMS에 종속

    LOG->>PHY: 릴레이션 스키마
    Note over PHY: 인덱스·저장 구조 설계\n접근 경로 설계\n물리적 구조 정의

    PHY->>IMP: 물리적 설계 명세서
    Note over IMP: SQL로 DB 생성\n트랜잭션 구현
```

## 이상(Anomaly)과 정규화 ★A

```mermaid
graph TD
    ANOM["⚠️ 이상 Anomaly\n불필요한 데이터 중복으로\n발생하는 부작용"]

    ANOM --> A1["삽입 이상\nInsertion Anomaly\n불필요한 데이터 함께\n삽입해야 하는 현상"]
    ANOM --> A2["삭제 이상\nDeletion Anomaly\n의도치 않은 데이터\n함께 삭제되는 현상"]
    ANOM --> A3["갱신 이상\nUpdate Anomaly\n일부만 갱신되어\n불일치 발생하는 현상"]

    ANOM -->|해결| NORM["📐 정규화 Normalization\n함수적 종속성을 이용해\n릴레이션을 분해"]

    NORM --> NF1["1NF\n원자값만 허용\n반복 그룹 제거"]
    NF1 --> NF2["2NF\n부분 함수 종속 제거\n기본키 전체에 완전 종속"]
    NF2 --> NF3["3NF\n이행 함수 종속 제거\nA→B, B→C이면 A→C 제거"]
    NF3 --> BCNF["BCNF\n결정자이면서 후보키가 아닌\n함수 종속 제거"]
    BCNF --> NF4["4NF\n다중값 종속 제거"]
    NF4 --> NF5["5NF\n조인 종속성 이용\n모든 조인 종속 제거"]

    style ANOM fill:#f8d7da
    style NORM fill:#d4edda
    style NF1 fill:#e8f4fd
    style NF2 fill:#e8f4fd
    style NF3 fill:#e8f4fd
    style BCNF fill:#fff3cd,stroke:#f0c000
```
