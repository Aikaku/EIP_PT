# 11장 응용 SW 기초 기술 활용 — 예상 문제

## 유형 1. 설명 → 용어 맞추기

> 설명을 읽고 해당하는 용어를 쓰시오.

**Q01.**  
컴퓨터 시스템의 자원들을 효율적으로 관리하며, 사용자가 컴퓨터를 편리하고 효과적으로 사용할 수 있도록 환경을 제공하는 여러 프로그램의 모임이다.

**Q02.**  
1960년대 AT&T 벨 연구소, MIT, General Electric이 공동 개발한 운영체제로, 시분할 시스템을 위해 설계된 대화식 운영체제이다. 대부분 C 언어로 작성되어 이식성이 높으며 트리 구조의 파일 시스템을 갖는다.

**Q03.**  
UNIX의 가장 핵심적인 부분으로 하드웨어를 보호하고, 프로그램과 하드웨어 간의 인터페이스 역할을 담당하며 프로세스 관리, 기억장치 관리, 파일 관리 등 여러 기능을 수행한다.

**Q04.**  
UNIX에서 사용자의 명령어를 인식하여 프로그램을 호출하고 명령을 수행하는 명령어 해석기로, 시스템과 사용자 간의 인터페이스를 담당한다.

**Q05.**  
구글(Google)에서 개발한 리눅스 커널 기반의 개방형 모바일 운영체제로, 모든 코드가 공개된 개방형 소프트웨어이다. 자바와 코틀린으로 애플리케이션을 작성한다.

**Q06.**  
기억장치 배치 전략 중, 프로그램이나 데이터가 들어갈 수 있는 빈 영역 중에서 단편화를 가장 작게 남기는 분할 영역에 배치시키는 방법이다.

**Q07.**  
기억장치 배치 전략 중, 프로그램이나 데이터가 들어갈 수 있는 빈 영역 중에서 단편화를 가장 많이 남기는 분할 영역에 배치시키는 방법이다.

**Q08.**  
가상기억장치 구현 기법 중, 프로그램과 주기억장치의 영역을 동일한 크기로 나눈 후 나눠진 프로그램을 동일하게 나눠진 주기억장치 영역에 적재시켜 실행하는 기법이다.

**Q09.**  
가상기억장치 구현 기법 중, 프로그램을 배열이나 함수 등과 같은 논리적인 크기로 나눈 후 주기억장치에 적재시켜 실행하는 기법이다. 각 단위는 고유한 이름과 크기를 갖는다.

**Q10.**  
페이지 교체 알고리즘 중, 각 페이지가 주기억장치에 적재될 때마다 그때의 시간을 기억시켜 가장 먼저 들어와서 가장 오래 있었던 페이지를 교체하는 기법이다.

**Q11.**  
페이지 교체 알고리즘 중, 최근에 가장 오랫동안 사용하지 않은 페이지를 교체하는 기법이다.

**Q12.**  
페이지 교체 알고리즘 중, 사용 빈도가 가장 적은 페이지를 교체하는 기법이다.

**Q13.**  
프로세스가 실행되는 동안 주기억장치를 참조할 때 일부 페이지만 집중적으로 참조하는 성질이 있다는 이론으로, 국부성·지역성·구역성·국소성이라고도 한다.

**Q14.**  
프로세스가 일정 시간 동안 자주 참조하는 페이지들의 집합이다. 이를 주기억장치에 상주시킴으로써 페이지 부재 및 페이지 교체 현상이 줄어들어 프로세스의 기억장치 사용이 안정된다.

**Q15.**  
프로세스의 처리 시간보다 페이지 교체에 소요되는 시간이 더 많아지는 현상으로, 가상기억장치를 사용하는 시스템에서 자주 페이지 부재가 발생함으로써 나타나며 전체 시스템의 성능이 저하된다.

**Q16.**  
준비 상태에서 대기하고 있는 프로세스 중 하나가 프로세서를 할당받아 실행 상태로 전이되는 과정이다.

**Q17.**  
입·출력 작업이 완료되어 프로세스가 대기 상태에서 준비 상태로 전이되는 과정이다.

**Q18.**  
입·출력장치의 느린 처리 속도를 보완하고 다중 프로그래밍 시스템의 성능을 향상시키기 위해 입·출력할 데이터를 직접 입·출력장치에 보내지 않고 나중에 한꺼번에 처리하기 위해 디스크에 저장하는 과정이다.

**Q19.**  
시스템의 여러 자원을 할당받아 실행하는 프로그램의 단위 또는 프로세스 내에서의 작업 단위로, 프로세스의 일부 특성을 갖고 있기 때문에 경량(Light Weight) 프로세스라고도 한다.

**Q20.**  
CPU 스케줄링 기법 중, 준비상태 큐에서 실행 시간이 가장 짧은 프로세스에게 먼저 CPU를 할당하는 기법으로, 가장 적은 평균 대기 시간을 제공하는 최적 알고리즘이다.

**Q21.**  
CPU 스케줄링 기법 중, 대기 시간과 서비스(실행) 시간을 이용하여 우선순위를 계산하는 기법이다.  
우선순위 계산식 : **(대기 시간 + 서비스 시간) / 서비스 시간**

**Q22.**  
CPU 스케줄링 기법 중, 각 프로세스를 시간 할당량(Time Slice, Quantum) 동안만 실행한 후 실행이 완료되지 않으면 다음 프로세스에게 CPU를 넘겨주는 기법이다. 시분할 시스템을 위해 고안된 방식이다.

**Q23.**  
CPU 스케줄링 기법 중, 현재 실행 중인 프로세스의 남은 시간과 준비상태 큐에 새로 도착한 프로세스의 실행 시간을 비교하여 가장 짧은 실행 시간을 요구하는 프로세스에게 CPU를 할당하는 기법이다.

**Q24.**  
UNIX 명령어 중, 파일의 보호 모드를 설정하여 파일의 사용 허가를 지정하는 명령어로, 8진법 숫자를 이용한 방법으로도 파일의 보호 모드를 설정할 수 있다.

**Q25.**  
UNIX/LINUX 명령어 중, PID(프로세스 고유 번호)를 이용하여 프로세스를 종료하는 명령어이다.

**Q26.**  
UNIX/LINUX 명령어 중, 현재 실행 중인 프로세스를 표시하는 명령어이다.

**Q27.**  
UNIX/LINUX 명령어 중, 파일이나 디렉터리의 초기 권한을 설정할 때 사용하는 값으로, 파일의 경우 666에서 이 값을 뺀 값을, 디렉터리의 경우 777에서 이 값을 뺀 값을 초기 접근 권한으로 갖는다.

**Q28.**  
IPv4의 주소 부족 문제를 해결하기 위해 개발된 IP 주소 체계로, 16비트씩 8부분 총 128비트로 구성되어 있다. 각 부분을 16진수로 표현하고 콜론(:)으로 구분한다.

**Q29.**  
IPv6의 주소 체계 중, 단일 송신자와 다중 수신자 간의 통신(1 대 다 통신)에 사용되는 방식이다.

**Q30.**  
IPv6의 주소 체계 중, 단일 송신자와 가장 가까이 있는 단일 수신자 간의 통신(1 대 1 통신)에 사용되는 방식이다.

**Q31.**  
다른 시스템 간의 원활한 통신을 위해 ISO(국제표준화기구)에서 제안한 통신 규약으로, 7계층으로 구성된 모델이다.

**Q32.**  
OSI 참조 모델 중, 서로 다른 데이터 표현 형태를 갖는 시스템 간의 상호 접속을 위해 코드 변환, 데이터 암호화, 데이터 압축, 구문 검색 등의 기능을 수행하는 계층이다.

**Q33.**  
네트워크 관련 장비 중, LAN과 LAN의 연결 기능에 데이터 전송의 최적 경로를 선택하는 기능이 추가된 장치로, 서로 다른 LAN 또는 LAN과 WAN을 연결하는 기능도 한다.

**Q34.**  
네트워크 관련 장비 중, OSI 전 계층의 프로토콜 구조가 다른 네트워크를 연결하는 장치로, LAN에서 다른 네트워크에 데이터를 보내거나 받아들이는 출입구 역할을 한다.

**Q35.**  
서로 다른 기기들 간의 데이터 교환을 원활하게 수행할 수 있도록 표준화시켜 놓은 통신 규약이다. 기본 요소로 구문(Syntax), 의미(Semantics), 시간(Timing)이 있다.

**Q36.**  
패킷 교환 방식 중, 단말기 상호 간에 논리적인 가상 통신 회선을 미리 설정하여 연결을 확립한 후 설정된 경로를 따라 패킷들을 순서적으로 운반하는 방식이다. 모든 패킷은 같은 경로로 발생 순서대로 전송된다.

**Q37.**  
패킷 교환 방식 중, 연결 경로를 설정하지 않고 인접한 노드들의 트래픽 상황을 감안하여 각각의 패킷들을 순서에 상관없이 독립적으로 운반하는 방식이다. 수신 측에서 패킷의 순서를 재정리한다.

**Q38.**  
인터넷에 연결된 서로 다른 기종의 컴퓨터들이 데이터를 주고받을 수 있도록 하는 표준 프로토콜이다.

**Q39.**  
TCP/IP에서 OSI 7계층의 전송 계층에 해당하며, 가상 회선 방식을 기반으로 하는 양방향 연결 서비스를 제공한다. 패킷의 다중화, 순서 제어, 오류 제어, 흐름 제어 기능을 제공한다.

**Q40.**  
TCP/IP에서 OSI 7계층의 네트워크 계층에 해당하며, 데이터그램 방식을 기반으로 하는 비연결형 서비스를 제공한다. 패킷의 분해/조립, 주소 지정, 경로 선택 기능을 제공한다.

**Q41.**  
데이터 전송 전에 연결을 설정하지 않는 비연결형 서비스를 제공하는 프로토콜로, TCP에 비해 오버헤드가 적고 전송 속도가 빠르다. 실시간 전송에 유리하며 신뢰성보다는 속도가 중요시되는 네트워크에서 사용된다.

**Q42.**  
호스트의 IP 주소를 호스트와 연결된 네트워크 접속 장치의 물리적 주소(MAC Address)로 변환하는 기능을 수행하는 프로토콜이다.

**Q43.**  
ARP와 반대로 물리적 주소를 IP 주소로 변환하는 기능을 수행하는 프로토콜로, 역순 주소 결정 프로토콜이라 불린다.

**Q44.**  
네트워크 관련 신기술 중, P2P 네트워크를 이용하여 온라인 금융 거래 정보를 온라인 네트워크 참여자의 디지털 장비에 분산 저장하는 기술이다.

**Q45.**  
네트워크 관련 신기술 중, 한 번의 로그인으로 개인이 가입한 모든 사이트를 이용할 수 있게 해주는 시스템이다.

**Q46.**  
네트워크 구축 형태 중, 모든 지점의 컴퓨터와 단말장치를 서로 연결한 형태로, 노드의 연결성이 높다. 노드 수가 n개일 때 n(n-1)/2개의 회선이 필요하다.

**Q47.**  
한 개의 정식 IP 주소에 대량의 가상 사설 IP 주소를 할당 및 연결하는 기능으로, 네트워크 주소 변환이라고도 한다.

**Q48.**  
IGP 중, 현재 가장 널리 사용되는 라우팅 프로토콜로 거리 벡터 라우팅 프로토콜이라고도 불리며, 최단 경로 탐색에 Bellman-Ford 알고리즘을 사용한다.

**Q49.**  
IGP 중, RIP의 단점을 해결하여 대규모 네트워크에서 많이 사용되며, 최단 경로 탐색에 다익스트라(Dijkstra) 알고리즘을 사용한다.

**Q50.**  
흐름 제어 방식 중, 확인 신호 없이도 미리 정해진 패킷의 수만큼 연속적으로 전송하는 방식으로, 한 번에 여러 개의 패킷을 전송할 수 있어 전송 효율이 좋다.

**Q51.**  
SW 관련 신기술 중, 웹에서 제공하는 정보 및 서비스를 이용하여 새로운 소프트웨어나 서비스, 데이터베이스 등을 만드는 기술로, 다수의 정보원이 제공하는 콘텐츠를 조합하여 하나의 서비스로 제공하는 웹 사이트 또는 애플리케이션을 말한다.

**Q52.**  
2개 이상의 하드디스크로 디스크 배열을 구성하고 파일을 구성하는 데이터 블록들을 서로 다른 디스크에 분산 저장하거나 다중화하는 저장 기술로, 구현된 기술에 따라 레벨로 구분한다.

**Q53.**  
RAID 0의 별칭이다. 디스크를 병렬로 연결하여 디스크의 개수만큼 용량과 속도가 배로 증가하지만 하나의 디스크만 손상되어도 전체 데이터가 파손된다.

**Q54.**  
RAID 1의 별칭이다. 같은 데이터를 다른 디스크에 동일하게 복사하는 방식이다.

**Q55.**  
트랜잭션들을 수행하는 도중 장애가 발생하여 데이터베이스가 손상되었을 때 손상되기 이전의 정상 상태로 복구하는 작업이다. 기법의 종류로 연기 갱신 기법, 즉각 갱신 기법, 그림자 페이지 대체 기법, 검사점 기법이 있다.

**Q56.**  
트랜잭션이 데이터를 갱신하면 트랜잭션이 부분 완료되기 전이라도 즉시 실제 데이터베이스에 반영하는 방법으로, Redo와 Undo 모두 사용 가능하다.

**Q57.**  
트랜잭션들이 어떤 로킹 단위를 액세스하기 전에 Lock(잠금)을 요청해서 Lock이 허락되어야만 그 로킹 단위를 액세스할 수 있도록 하는 기법으로, 주요 데이터의 액세스를 상호 배타적으로 한다.

**Q58.**
