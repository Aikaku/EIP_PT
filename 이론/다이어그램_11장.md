## 프로세스 상태 전이 ★A

```mermaid
stateDiagram-v2
    [*] --> 준비상태: 프로세스 생성

    준비상태 --> 실행상태: Dispatch\n(스케줄러가 CPU 할당)

    실행상태 --> 준비상태: Time Runout\n(시간 초과, 선점)

    실행상태 --> 대기상태: Block\n(I/O 요청·이벤트 대기)

    대기상태 --> 준비상태: Wake Up\n(I/O 완료·이벤트 발생)

    실행상태 --> [*]: 종료
```

> - **Dispatch**: 준비 → 실행 (CPU 할당)
> - **Wake Up**: 대기 → 준비 (I/O 완료)
> - **Spooling**: 입출력과 CPU 처리를 중간 버퍼(스풀)로 병행 처리

## 스케줄링 알고리즘 ★A

```mermaid
graph TD
    SCHED["⏰ 스케줄링 CPU 스케줄링\nCPU를 어떤 프로세스에 할당할지 결정"]

    SCHED --> NP["비선점 Non-Preemptive\n실행 중 다른 프로세스가 CPU 빼앗지 못함"]
    SCHED --> P["선점 Preemptive\n실행 중 다른 프로세스가 CPU 차지 가능"]

    NP --> FCFS["FCFS First Come First Served\n도착 순서대로 처리\n=FIFO 큐"]
    NP --> SJF["SJF ★ Shortest Job First\n실행 시간 짧은 것 먼저\n기아 현상 가능"]
    NP --> HRN["HRN ★\nHighest Response-Ratio Next\nSJF 보완, 대기시간 고려\n우선순위 = (대기+서비스)/서비스\n값이 클수록 우선"]

    P --> RR["RR ★ Round Robin\n시분할 = 일정 시간(퀀텀)씩 순환\n퀀텀 크면 FCFS, 작으면 오버헤드"]
    P --> SRT["SRT Shortest Remaining Time\nSJF의 선점 버전\n남은 시간 짧은 것 선점"]
    P --> MFQ["다단계 피드백 큐 MFQ\n여러 큐에 우선순위\n하위로 이동하며 처리"]

    style SCHED fill:#fff3cd
    style NP fill:#cce5ff
    style P fill:#d4edda
    style SJF fill:#cce5ff,stroke:#004085
    style HRN fill:#cce5ff,stroke:#28a745
    style RR fill:#d4edda,stroke:#28a745
```

> **HRN 우선순위 공식**: (대기시간 + 서비스시간) / 서비스시간
